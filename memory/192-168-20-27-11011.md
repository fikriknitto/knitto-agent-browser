# Knitto Monitoring Incident (http://192.168.20.27:11011)

## Login
- URL: http://192.168.20.27:11011 → klik Login di header → /login
- Fields: placeholder `Username`, `Password`; tombol `LOGIN`
- Sukses: redirect ke `/admin`, tombol `admin` dan `Log out` terlihat

## Navigasi
- Hamburger: Icon button kiri atas (gunakan clickCenter:true)
- Menu Incidents: menuitem `Incidents` → `/admin/incidents`

## Tambah Incident
- Tombol: `Tambah Incident` membuka modal form
- Fields: Judul (`Masukkan judul incident`), Deskripsi (contenteditable div), Severity/Status (custom dropdown), Tanggal & Waktu (opsional), Service + Status Service per baris
- Custom dropdown: klik field → `automation_get_page_snapshot` → cari opsi yang teksnya **sama** atau **mengandung** nilai target → `automation_click` baris/option tersebut (atau `automation_click_at` pada bbox center jika click gagal)
- Dropdown Service dengan search: buka field → isi `Cari service` dengan kata kunci (mis. "Manpro Pusat") → tunggu filter → klik baris hasil yang cocok/mengandung teks target
- Service options: Order Pusat, Take Order, Manpro Pusat, Server Staging Yogya
- Status Service: Operational, Degraded Performance, Partial Outage, Major Outage, Maintenance
- Simpan: tombol `Simpan` di bawah modal; modal tertutup otomatis setelah sukses
- Jangan gunakan Escape untuk menutup modal

## Catatan teknis
- Saat dropdown terbuka: prioritaskan cari teks exact match, lalu partial contains, baru klik opsi — jangan tebak koordinat tanpa snapshot
- `automation_click` dengan `{ text: "..." }` atau `{ role: "option", name: "..." }` sering gagal pada dropdown custom React; fallback `automation_click_at` pada bbox dari snapshot
- `automation_select_option` tidak cocok untuk dropdown custom React
- Field Service wajib diisi sebelum Simpan berhasil
